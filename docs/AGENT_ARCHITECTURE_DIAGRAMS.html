Diagram 1

The agent loop

This is the fundamental mechanic. Every user message triggers this loop. The loop only ends when Claude produces a response with no tool_use blocks — until then, it keeps calling tools and re-calling itself.

  • Purple boxes: Claude API calls (Sonnet 4.6)
  • Teal box: tool execution — happens in Python, not in the model
  • The loop can call many tools per turn before returning a response
  • The messages array grows each turn — that's session memory

Diagram 2

Classification use case — human-in-the-loop

When the user asks to classify unknown transactions, the agent calls three tools before asking for human approval. The write operation only happens after the human explicitly approves.

  • The suggest step uses Haiku 4.5 — a separate internal API call, not the main loop model
  • Preview is a read-only operation — the database is not touched until the human says yes
  • On rejection, the agent refines the suggestion and previews again

Diagram 4

Data architecture — three stores, two lifespans

The agent reads from and writes to three distinct stores on every turn. The key distinction is lifespan: session memory resets every conversation, while agent state and transaction data persist forever.

Session memory — resets per conversation
Agent state — persists across sessions (SQLite)
Transaction data — source of truth (SQLite)

Key insight

What gets built on each API call

Every single API call assembles context from all three stores. Understanding this is understanding the whole agent.

messages = [ {"role": "system", "content": SYSTEM_PROMPT}, ← static {"role": "system", "content": agent_state_snapshot}, ← from SQLite {"role": "user", "content": first message}, ← session turn 1 {"role": "assistant","content": "...[tool calls]..."}, ← session turn 1 {"role": "user", "content": "tool results"}, ← injected by loop {"role": "user", "content": second message}, ← session turn 2 ... ]

The agent never "remembers" — it re-reads state on every call. The session history is always replayed in full. The agent_state snapshot gives it cross-session continuity without replaying old sessions.
